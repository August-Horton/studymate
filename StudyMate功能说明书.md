# StudyMate 智学助手 - 功能说明书

## 一、产品概述

### 1.1 产品简介

**StudyMate 智学助手**是一款基于人工智能技术的智能学习辅助工具，旨在帮助学生高效管理学习内容、制定科学的备考计划、提升学习效率。本产品采用现代化的技术架构，结合先进的学习科学理论，为用户提供全方位的学习支持。

### 1.2 产品定位

- **目标用户**：学生群体（中学生、大学生）、自学者、备考人员
- **核心价值**：智能化学习管理、个性化备考规划、高效知识复习
- **产品口号**：让学习更智慧，让备考更轻松

### 1.3 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                    用户界面层 (UI Layer)                    │
│  Vue 3 + Vant 4 + Electron (桌面应用)                        │
├─────────────────────────────────────────────────────────────┤
│                    业务逻辑层 (Service Layer)               │
│  API 接口调用、状态管理、本地缓存                             │
├─────────────────────────────────────────────────────────────┤
│                    后端服务层 (Backend Layer)               │
│  FastAPI + SQLAlchemy + SQLite                             │
├─────────────────────────────────────────────────────────────┤
│                    人工智能层 (AI Layer)                    │
│  DeepSeek API + 自定义学习算法                              │
└─────────────────────────────────────────────────────────────┘
```

### 1.4 主要功能模块

| 模块名称 | 功能描述 | 状态 |
|---------|---------|------|
| 笔记闪卡 | 笔记管理、闪卡生成、知识图谱 | ✅ 已完成 |
| 备考计划 | 智能诊断、个性化计划、进度追踪 | ✅ 已完成 |

## 二、功能详细说明

### 2.1 笔记闪卡模块

#### 2.1.1 功能概述

笔记闪卡模块是 StudyMate 的核心功能之一，提供完整的知识管理解决方案，支持笔记创建、分类整理、智能闪卡生成等功能。

#### 2.1.2 核心功能

##### 2.1.2.1 知识库管理

**功能描述**：用户可以创建、编辑、删除笔记和文件夹，支持树形目录结构管理。

**功能特性**：
- 支持多级文件夹结构
- 笔记标题和内容编辑
- 笔记搜索功能
- 笔记标签分类

**操作流程**：
1. 用户进入笔记管理页面
2. 点击「新建笔记」按钮
3. 输入笔记标题和内容
4. 选择保存位置（文件夹）
5. 点击保存完成创建

##### 2.1.2.2 闪卡生成

**功能描述**：基于笔记内容自动生成闪卡，采用 SM-2 记忆算法进行智能复习。

**功能特性**：
- 自动从笔记中提取知识点
- 智能生成问题-答案对
- 支持手动编辑闪卡
- 基于遗忘曲线的复习提醒

**技术实现**：
```
笔记内容 → AI分析 → 知识点提取 → 闪卡生成 → 复习计划
```

##### 2.1.2.3 知识图谱

**功能描述**：可视化展示知识点之间的关联关系，帮助用户建立知识网络。

**功能特性**：
- 自动分析笔记间的关联
- 图形化展示知识结构
- 支持节点点击跳转
- 交互式缩放和拖拽

##### 2.1.2.4 笔记导入导出

**功能描述**：支持从外部导入 Markdown 文件，也可将笔记导出备份。

**功能特性**：
- 支持批量导入文件夹
- 保留原有目录结构
- 导出为 Markdown 格式
- 支持图片等附件

#### 2.1.3 界面设计

**侧边栏布局**：
- 知识库标题和操作按钮
- 目录/大纲/闪卡选项卡切换
- 搜索框
- 文件夹树结构

**主编辑区布局**：
- Markdown 编辑器
- 实时预览
- 工具栏（格式化、插入图片等）
- 大纲导航

#### 2.1.4 技术实现

**前端组件**：
- `Notes.vue` - 主页面组件
- `FolderTree.vue` - 文件夹树组件
- `MarkdownViewer.vue` - Markdown 预览组件
- `GraphView.vue` - 知识图谱组件

**后端 API**：
| API 路径 | 方法 | 功能 |
|---------|------|------|
| `/api/notes/` | GET | 获取笔记列表 |
| `/api/notes/{id}` | GET | 获取单条笔记 |
| `/api/notes/` | POST | 创建笔记 |
| `/api/notes/{id}` | PUT | 更新笔记 |
| `/api/notes/{id}` | DELETE | 删除笔记 |
| `/api/notes/search` | GET | 搜索笔记 |

### 2.2 备考计划模块

#### 2.2.1 功能概述

备考计划模块利用人工智能技术，为用户制定个性化的学习计划，通过多轮诊断评估学习状态，生成针对性的复习方案。

#### 2.2.2 核心功能

##### 2.2.2.1 多轮诊断系统

**功能描述**：通过三轮诊断评估，全面了解用户的学习状况和薄弱环节。

**诊断流程**：
- **第一轮**：基础信息收集（课程、考试日期、目标分数）
- **第二轮**：知识掌握度评估（选择题形式）
- **第三轮**：学习习惯和时间安排调查

**技术实现**：
```
用户输入 → AI分析 → 生成诊断问题 → 用户作答 → 智能评估 → 生成报告
```

##### 2.2.2.2 智能计划生成

**功能描述**：基于诊断结果，自动生成个性化的备考计划。

**计划特性**：
- 根据剩余天数分配学习任务
- 优先安排薄弱知识点
- 合理分配每日学习时间
- 包含复习和练习任务

**算法逻辑**：
1. 分析诊断数据，识别薄弱环节
2. 根据考试日期计算剩余天数
3. 分配每天的学习任务量
4. 生成每日详细计划

##### 2.2.2.3 进度追踪

**功能描述**：实时追踪学习进度，提供可视化的进度展示。

**进度展示**：
- 备考火力值进度条
- 每日任务完成情况
- 打卡热力图
- 任务统计数据

**激励机制**：
- 连续打卡奖励
- 进度达成提醒
- 成就徽章系统

##### 2.2.2.4 紧急救援计划

**功能描述**：当用户进度落后时，自动生成紧急补救计划。

**救援机制**：
- 智能检测进度落后情况
- 生成针对性补救方案
- 优先突击薄弱知识点
- 优化时间分配策略

#### 2.2.3 界面设计

**信息录入区**：
- 课程选择
- 年级选择
- 考试日期选择
- 目标分数输入

**诊断问卷区**：
- 问题展示
- 选项选择（单选/多选/文本）
- 进度指示
- 提交按钮

**计划展示区**：
- 备考建议卡片
- 进度仪表盘
- 每日任务列表
- 热力图

#### 2.2.4 技术实现

**前端组件**：
- `Plans.vue` - 主页面组件
- `CourseSelector.vue` - 课程选择器组件

**后端 API**：
| API 路径 | 方法 | 功能 |
|---------|------|------|
| `/api/plans/questionnaire` | POST | 生成诊断问卷 |
| `/api/plans/answers` | POST | 提交答案 |
| `/api/plans/detail` | POST | 生成详细计划 |
| `/api/plans/daily` | POST | 生成每日计划 |
| `/api/plans/rescue` | POST | 生成救援计划 |

## 八、故障排除

### 8.1 常见问题

**问题1：应用启动后白屏**
- 可能原因：前端资源加载失败
- 解决方案：检查网络连接，尝试刷新页面

**问题2：课程数据丢失**
- 可能原因：数据库路径不一致
- 解决方案：确保使用统一的数据库位置

**问题3：后端连接失败**
- 可能原因：后端服务未启动
- 解决方案：检查后端服务状态，尝试重启

**问题4：笔记无法保存**
- 可能原因：数据库写入权限不足
- 解决方案：检查数据库文件权限

### 8.2 日志与调试

**日志位置**：
